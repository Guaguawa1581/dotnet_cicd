# 系統安裝說明

## 前置需求

- 安裝 [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- Windows 10/11 或 Linux

---

## 安裝步驟

### 1. 確認收到以下檔案

```
app-bundle.tar
docker-compose.yml
.env.example
安裝說明.md
```

### 2. 建立環境設定檔

將 `.env.example` 複製並重新命名為 `.env`：

```bash
cp .env.example .env
```

以文字編輯器開啟 `.env`，填入實際的資料庫連線資訊：

```
DB_CONNECTION_STRING=Server=你的DB位址; Database=資料庫名稱; User Id=帳號; Password=密碼; TrustServerCertificate=True;
APP_PORT=5088
```

### 3. 載入 Docker Image

開啟終端機（PowerShell 或 CMD），進入檔案所在目錄：

```bash
docker load -i app-bundle.tar
```

### 4. 啟動系統

```bash
docker compose up -d
```

### 5. 確認是否正常運作

開啟瀏覽器，進入：

```
http://localhost:5088/WeatherForecast
```

看到 JSON 資料代表系統啟動成功。

## 常用指令

| 動作 | 指令 |
|------|------|
| 啟動 | `docker compose up -d` |
| 停止 | `docker compose down` |
| 查看狀態 | `docker compose ps` |
| 查看錯誤 | `docker compose logs api` |

---

## 更新系統

收到新版 `app-bundle.tar` 後：

```bash
# 1. 停止目前服務
docker compose down

# 2. 載入新 image
docker load -i app-bundle.tar

# 3. 重新啟動
docker compose up -d
```

---

## 常見問題

**Q: 無法連線到資料庫？**  
A: 確認 `.env` 裡的 `DB_CONNECTION_STRING` 位址與帳密是否正確。

**Q: port 5088 被佔用？**  
A: 修改 `.env` 裡的 `APP_PORT` 為其他 port，例如 `APP_PORT=8080`。

**Q: 如何確認容器有在跑？**  
A: 執行 `docker compose ps`，狀態應顯示 `running`。
